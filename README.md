## Вариант 4: Простой шаблонизатор
###  Важно! Решение возможной проблемы со сборкой (Ошибка "version 69")

На некоторых компьютерах, где установлено несколько версий Java (особенно "preview" или "early access" версии), сборка может провалиться с ошибкой `Unsupported class file major version 69`.

Это **проблема не проекта, а настроек Java на конкретном компьютере**. Чтобы это исправить, нужно перед сборкой явно указать, какую версию Java использовать.

**Выполните эти шаги в командной строке Windows (`cmd.exe`) ПЕРЕД тем, как переходить к Шагу 1 ниже.**

1.  **Найдите, где у вас установлена правильная Java.**
    Обычно это папка `C:\Program Files\Java\`. Внутри нее найдите папку с нужной версией, например `jdk-17.0.8`. Скопируйте этот путь.

2.  **Откройте командную строку (cmd)** и выполните следующие две команды. Они установят правильный путь только для текущего окна терминала, ничего не сломав в системе.

    *   **Устанавливаем `JAVA_HOME` (замените путь на свой):**
        ```cmd
        set JAVA_HOME="C:\Program Files\Java\jdk-17.0.8"
        ```

    *   **Добавляем этот путь в начало системного PATH (эту команду не меняйте):**
        ```cmd
        set PATH=%JAVA_HOME%\bin;%PATH%
        ```

3.  **Проверьте, что все сработало.** Версия Java должна быть той, которую вы указали.
    ```cmd
    java -version
    ```

**Теперь, в этом же окне командной строки, можно переходить к Шагу 1 и выполнять все остальные команды.**

---

###  1: Проверка на наличие нужных компонентов

Прежде чем начать, убедитесь, что у вас на компьютере есть две вещи:

1.  **Java Development Kit (JDK) версии 17.**
    *   *Как проверить?* Открой терминал и напиши `java -version`. Если выдаст ошибку, то нужно установить.
2.  **Git.**
    *   *Как проверить?* В том же терминале напиши `git --version`. Если есть ответ — все хорошо.

> **Важно:** Тебе **НЕ НУЖНО** устанавливать Gradle. В проекте есть штука под названием **Gradle Wrapper** (`gradlew`), которая сама скачает и использует нужную версию Gradle.

###  2. Скачивание и сборка проекта

Все команды нужно вводить в терминале **Git Bash** (или в том же окне `cmd`, где вы настраивали `JAVA_HOME`).

1.  **Клонируй (скачай) репозиторий с GitHub.** Эта команда создаст на твоем компьютере папку `simple-templater`.
    ```bash
    git clone https://github.com/vladeess/simple-templater.git
    ```

2.  **Перейди в созданную папку.**
    ```bash
    cd simple-templater
    ```

3.  **Запусти сборку.** Эта команда сделает все за тебя: скачает зависимости, скомпилирует код, запустит тесты и упакует все в один исполняемый `.jar` файл.
    ```bash
    ./gradlew build

    ```
**или**
```bash
     gradlew build
```

Дождись, пока в конце не появится зеленая надпись **`BUILD SUCCESSFUL`**. Если появилась — все прошло отлично.

###  3. Настройка проекта перед запуском

Чтобы приложение знало, какой шаблон и какие данные использовать, ему нужен конфигурационный файл.

1.  В корневой папке проекта (`simple-templater`) создай главный файл `app.properties`:
    ```bash
    touch app.properties

    ```
**или**

```bash
     type nul > app.properties

```


2.  Открой этот файл `app.properties` блокнотом и вставь в него следующий текст:
    ```properties
    # Путь к файлу шаблона
    template.file.path=./examples/template.txt

    # Путь к файлу с данными
    data.file.path=./examples/data.properties

    # Формат данных: PROPERTIES или JSON
    data.format=PROPERTIES
    ```

###  4. Запуск проекта

Теперь, когда все собрано и настроено, запускаем приложение.

1.  Выполни эту команду в терминале.
    ```bash
    java -jar build/libs/simple-templater-1.0.0.jar

    ```
**или**
```bash
     java -jar build\libs\simple-templater-1.0.0.jar

```


2.  **Готово!** В консоли должен появиться результат работы:
    ```
    Hello, Bob!
    Your order A-123 is confirmed.
    Total: $99.50
    ```
